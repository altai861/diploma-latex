\section{Системийн архитектур}
\paragraph{}
Дараах зурагт хувийн үүлэн хадгалалтын системийн архитектур диаграммыг харуулав. Энгийн хэрэглэгч болон админ гэсэн 2 төрлийн хэрэглэгч байх агаад бүх хэрэглэгчид веб олон IOS апп ашиглан системтэй харьцана. Веб апп-ийг Angular, IOS апп-ийг Swift технелогиудыг ашиглан хөгжүүлнэ. Тухайн хэрэглэгчийн апп-ууд нь сервертэй HTTPS холболтоор харьцаж зорилтот үйлдлүүдийг гүйцэтгэдэг байна. Сервер апп нь хэрэглэгчдийн нэвтрэлт, баталгаажуулалт, хэрэглэгчийн менежмент, файл менежмент, файл татах болон байршуулах үйлдлүүдийг хийнэ. Системийн нийт өгөгдлийг Relational өгөгдлийн бааз болон серверийн File system-д хадгална. Тухайн үүлэн хадгалалт дээр байрших файлууд болон хавтаснуудын мета өгөгдлүүд нь өгөгдлийн бааз дээр хадгалагдана.

\section{Зохиомжийн үндсэн компонентууд}
\paragraph{}
Системийн зохиомжид дараах үндсэн компонентуудыг тодорхойлсон.
\begin{enumerate}[nosep]
    \item \textbf{Хэрэглэгчийн удирдлагын модуль} -- нэвтрэх, нууц үг шалгах, session/token удирдах.
    \item \textbf{Файл удирдлагын модуль} -- хавтас үүсгэх, файл байршуулах, татах, нэр өөрчлөх, устгах.
    \item \textbf{Хуваалцах эрхийн модуль} -- хэрэглэгч болон бүлгийн түвшинд read/edit эрх олгох.
    \item \textbf{Storage abstraction} -- локал диск дээрх өгөгдөл болон өгөгдлийн сангийн мета мэдээллийг уялдуулах.
\end{enumerate}

\section{Өгөгдлийн ерөнхий схем}
\paragraph{}
Өгөгдлийн сан нь системийн ажиллагаанд шаардлагатай мета мэдээллийг хадгална. Үндсэндээ хэрэглэгч, файл/хавтас, эрхийн хамаарал гэсэн бүтэцтэй бөгөөд том хэмжээний бинар өгөгдөл нь серверийн файл систем дээр хадгалагдаж, өгөгдлийн санд зөвхөн зам, хэмжээ, эзэмшигч, хугацаа, эрхийн мэдээллүүд бүртгэгдэнэ.

\section{Системийн прототип}
\paragraph{}
Системийн ерөнхий интерфейсийн прототипийг Figma орчинд бэлтгэсэн. Прототип нь эхний тохиргоо, нэвтрэх хуудас, энгийн хэрэглэгчийн нүүр хуудас, админ хэрэглэгчийн нүүр хуудас гэсэн үндсэн урсгалуудыг хамарсан.

\section{Хөгжүүлсэн системийн интерфейс}
\paragraph{}
Хөгжүүлсэн интерфейс нь прототипийн үндсэн урсгалыг хадгалсан бөгөөд дараах дарааллаар ажиллана.
\begin{enumerate}[nosep]
    \item Анхны тохиргоогоор админ хэрэглэгч болон хадгалалтын үндсэн хавтас үүсгэнэ (Зураг \ref{fig:admin-first-launch}).
    \item Хэрэглэгч серверийн хаяг, хэрэглэгчийн нэр, нууц үгээр нэвтэрнэ (Зураг \ref{fig:login-screen}).
    \item Энгийн хэрэглэгч файлуудаа удирдаж, хуваалцсан агуулгад хандана (Зураг \ref{fig:normal-user-homepage}).
    \item Админ хэрэглэгч хэрэглэгч болон системийн ерөнхий тохиргоог удирдана (Зураг \ref{fig:admin-home-page}).
\end{enumerate}

\section{Системд хийгдсэн тест}
\paragraph{}
Системийн баталгаажуулалтыг дараах түвшинд гүйцэтгэсэн.
\begin{enumerate}[nosep]
    \item \textbf{Функциональ тест}: Нэвтрэх, файл байршуулах, татах, нэр өөрчлөх, устгах, эрх олгох урсгалуудыг usecase тус бүрээр шалгасан.
    \item \textbf{Эрхийн тест}: Админ болон энгийн хэрэглэгчийн эрх ялгарах, зөвшөөрөлгүй хандалтыг блоклох шалгалт хийсэн.
    \item \textbf{Интерфейсийн тест}: Веб болон мобайл клиент дээр үндсэн дэлгэцүүдийн хэрэглээний урсгал, алдааны мэдэгдлийг шалгасан.
\end{enumerate}

\section{Системийн нэвтрүүлэлт}
\paragraph{}
Системийг дотоод туршилтын орчинд self-hosted хэлбэрээр нэвтрүүлж, серверийн локал storage directory-т өгөгдөл хадгалах байдлаар ажиллуулсан. Нэвтрүүлэлтийн үе шатанд хэрэглэгчийн бүртгэл үүсгэх, файл байршуулах, эрх хуваалцах, татах урсгалуудыг бодит хэрэглээний сценариар туршиж, ажиллагааны алдаануудыг засварласан.


\section{Бүлгийн дүгнэлт}
\paragraph{}
Энэ бүлэгт хувийн үүлэн хадгалалтын системийн архитектур, үндсэн компонентууд, өгөгдлийн зохиомж, прототип болон хэрэгжүүлсэн интерфейсийг тайлбарлалаа. Системийн шийдэл нь хэрэглэгчийн өгөгдлийг өөрийн сервер дээр хадгалах self-hosted зарчимд тулгуурласан бөгөөд веб болон мобайл клиентээр дамжуулан нэгдсэн API-р ажиллахаар зохион байгуулагдсан. Мөн функцийн болон эрхийн түвшний тестүүдээр системийн үндсэн урсгалуудыг баталгаажуулсан.
