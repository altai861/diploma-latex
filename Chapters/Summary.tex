\section*{Ерөнхий дүгнэлт}

% Төслийн ерөнхий явц хэр байсан, Төслийн менежментийн хэрхэн гүйцэтгэсэн, кодын менежментыг хэрхэн гүйцэтгэсэн, кодыг хаана байршуулсан, Системийг хаана байршуулж ажиллуулж байгаа, Цаашид системийг хаана байршуулж нийтийн хүртээл болгох, Системийг бодит хэрэглэгч дээр ямар хэмжээнд нэвтрүүлсэн, альфа бета тестүүд ямар хэмжээнд хийгдсэн зэргийг дүгнэлтдээ оруулаарай. 


\paragraph{}
Энэхүү дипломын ажлын хүрээнд хэрэглэгч өөрийн өгөгдлөө өөрийн төхөөрөмж дээр хадгалж, веб болон мобайл орчноос ашиглах боломжтой self-hosted хувийн үүлэн хадгалалтын системийг судалж, зохиомжилж, хэрэгжүүлэв. Судалгааны шатанд Google Drive, Nextcloud, ownCloud зэрэг үүлэн хадгалалтын шийдлүүдийн архитектур, өгөгдөл хадгалалтын зарчим болон self-hosted системд тулгардаг үндсэн асуудлуудыг авч үзсэн. Үүний үндсэн дээр хэрэглэгчийн өгөгдлийн хяналт, бага нөөцтэй төхөөрөмж дээр ажиллах боломж, дотоод сүлжээнд шууд холбогдох чадвар болон гадаад сүлжээнээс хандах боломжийг системийн гол шаардлага болгон тодорхойлсон.

\paragraph{}
Шаардлага ба зохиомжийн шатанд системийн хэрэглэгчид, хэрэглээний тохиолдлууд, функциональ болон функциональ бус шаардлагуудыг тодорхойлж, файл, хавтас, хэрэглэгч, session, permission болон тохиргоотой холбоотой өгөгдлийн бүтцийг боловсруулсан. Системийг үндсэн сервер, дамжуулагч сервер, веб клиент болон iOS клиентээс бүрдэх архитектуртайгаар төлөвлөж, LAN-first холболт, mDNS-р сервер илрүүлэх боломж, NAT-ийн ард байрлах төхөөрөмжид relay server-ээр хандах шийдлийг тусгасан. Хэрэгжүүлэлтийн шатанд үндсэн серверийг Raspberry Pi 5 төхөөрөмж дээр байршуулж, PostgreSQL өгөгдлийн сан болон local filesystem хадгалалтыг ашигласан бол алсын хандалтад зориулсан дамжуулагч серверийг Hetzner VPS дээр Docker Compose ашиглан ажиллуулж, Caddy reverse proxy-р HTTPS/WSS холболтыг тохируулсан. Мөн домэйн нэрийг VPS-ийн public IP хаягтай холбож, веб болон iOS клиентүүдээс бодит орчинд системд хандах боломж бүрдүүлсэн.

\paragraph{}
Системийн ажиллагааг гар ажиллагаатай функциональ болон end-to-end туршилтаар шалгахад анхны тохиргоо, нэвтрэлт, файл байршуулах, татах, хавтас үүсгэх, нэр өөрчлөх, устгах, сэргээх, одлох, хуваалцах болон админ удирдлагын үндсэн урсгалууд веб болон мобайл клиент дээр ажиллаж байв. Мөн дотоод сүлжээнд Raspberry Pi рүү шууд хандах, гадаад сүлжээнээс дамжуулагч серверийн public domain хаягаар хандах хоёр хувилбар амжилттай туршигдсан. Иймээс энэхүү ажил нь self-hosted хувийн үүлэн хадгалалтын үндсэн боломжуудыг MVP түвшинд хэрэгжүүлэх боломжтойг харуулсан гэж дүгнэж байна. Хэрэглэгчийн файл болон metadata нь Raspberry Pi дээр хадгалагдаж, public сервер зөвхөн алсын холболтыг зуучлах үүрэгтэй ажилласан нь системийн үндсэн зорилготой нийцэж байна. Цаашид production түвшинд ашиглахын тулд end-to-end шифрлэлт, relay authentication, автомат тест, ачааллын туршилт, backup болон monitoring-ийн механизмыг сайжруулах шаардлагатай.
