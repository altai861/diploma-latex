\section{Үүлэн хадгалалтын системийн ерөнхий ойлголт}

\paragraph{}
Үүлэн хадгалалт (\textit{Cloud Storage}) нь өгөгдлийг локал төхөөрөмж дээр бус, сүлжээгээр холбогдсон алсын сервер дээр хадгалах технологи юм. Хэрэглэгч интернэтээр дамжуулан өөрийн файлуудад хаанаас ч, ямар ч төхөөрөмжөөс хандах боломжтой байдаг. Энэхүү технологи нь өгөгдлийг төвлөрсөн байдлаар хадгалах, нөөцлөх, хуваалцах зэрэг давуу талуудыг олгодог.

\paragraph{}
Үүлэн хадгалалтын системийг ерөнхийд нь дараах гурван төрөлд ангилдаг:

\begin{itemize}
    \item \textbf{Нийтийн (Public Cloud)} – Гуравдагч байгууллагын эзэмшлийн сервер дээр өгөгдлийг хадгалдаг (жишээ нь: Google Drive, Dropbox). Хэрэглэхэд хялбар боловч өгөгдлийн хяналт харьцангуй бага байдаг.
    
    \item \textbf{Хувийн (Private Cloud)} – Хэрэглэгч өөрийн сервер дээр системийг ажиллуулж, өгөгдлөө бүрэн хянах боломжтой. Аюулгүй байдал болон нууцлал өндөр боловч тохиргоо, удирдлага шаарддаг.
    
    \item \textbf{Холимог (Hybrid Cloud)} – Нийтийн болон хувийн үүлэн системийг хослуулсан хэлбэр. Уян хатан байдал сайтай боловч архитектурын хувьд илүү төвөгтэй.
\end{itemize}

\paragraph{}
Сүүлийн жилүүдэд өгөгдлийн нууцлал, аюулгүй байдлын асуудал нэмэгдэж байгаатай холбоотойгоор хувийн үүлэн хадгалалтын системд анхаарах хандлага нэмэгдэж байна. Хэрэглэгч өөрийн өгөгдлийг бүрэн хянах, гуравдагч этгээдээс хамааралгүй байх шаардлага нь ийм төрлийн системийг хөгжүүлэх үндэслэл болж байна. 2025 оны судалгаагаар \cite{cloudstorageusage} дэлхий дээр нийт 2.3 тэрбум хүн үүлэн хадгалалтын үйлчилгээг гуравдагч системээс авч байгаа гэсэн судалгаа гарсан. Үүний хэрэглэгчдийн ашиглалтын үзүүлэлт болон заах зээлийн хуваалт нь Google Drive 94\%, Dropbox 66\%, OneDrive 55\%, iCloud 50\% (хэрэглэгчид эдгээр платформуудыг зэрэг хэрэглэсэн байна). Дараах зурагт хэрэглээний түвшинг графикаар харуулав.

\paragraph{}
Судалгааны үр дүнгээс үзэхэд хэрэглэгчдийн 55\% дор хаяж 3 үүлэн хадгалалтын үйлчилгээг зэрэг хэрэглэж байна. Хүмүүс ихэвчлэн файлуудаа платформуудад хуваан, тухайн үйлчилгээний онцлогт тааруулан ашигладаг. Google Photos-ийг зураг хадгалахдаа ашиглаж, баримт бичгийг Dropbox, iPhone утасныхаа өгөгдлийг iCloud дээр хадгалах нь түгээмэл ажиглагдаж байна. Хэрэглэгчдийн дунд өгөгдлийн аюулгүй байдлын халдлага болон асуудал энэхүү салбарт хамгийн том асуудлуудын нэгт орж байгаа бөгөөд үүнд зориулж үйлчилгээ үзүүлэгч байгууллагууд илүү найдвартай шифрлэлт, хоёр шатлалт баталгаажуулалт гэх мэт хамгаалалтын шийдлүүдийг санал болгож байна. 2024 оны Англи улсад хэрэглэгчдээс авсан судалгаагаар үүлэн хадгалалтын хэрэглэгчдийн 43\% нь тухайн үүлэн хадгалалтын үйлчилгээ үзүүлэгч байгууллага нь өөрсдийнх нь хувийн өгөгдлийг шинжилдэг гэдэгт санаа зовнидог гэсэн хариулт өгсөн байна. Үүлэн хадгалалтын үйлчилгээнүүд нь ойрын ирээдүйд хиймэл оюунтай ашиглан файл системийн автомат цэгцлэлт, ухаалаг хайлт, өгөгдөл шошгожуулах, олон платформ дээр дэмжигдэх байдал, сүлжээгүй үед хэрхэн sync хийгдэх гэсэн зүйлс дээр нэмэлт сайжруулалт хийгдэн өрсөлдөөн үүсэх төлөвтэй байгаа ажээ.

\section{Ижил төст системийн судалгаа}
\paragraph{}
Хувийн өгөгдөл хадгалах, синхрончлох системүүд нь сүүлийн жилүүдэд хэрэглэгчийн өгөгдөлд бүрэн хяналттай байх, аюулгүй байдал болон cloud-ын энгийн хэрэглээний туршлагыг хадгалах чиглэлд эрчимтэй хөгжиж байна. Энэ хүрээнд уламжлалт төвлөрсөн cloud системүүд болон peer-to-peer системүүдийг судалгаанд оруулж байна.
\paragraph{}
Nextcloud нь self-hosted cloud storage системүүдийн түгээмэл жишээ бөгөөд сервер төвтэй архитектур ашиглан файл хадгалалт, синхрончлол, хуваалцах болон хамтын ажиллагааны олон төрлийн боломжуудыг нэг дор нэгтгэдэг. Nextcloud нь Universal File Access \cite{nextcloud2018architecture} давхаргаар дамжуулан төрөл бүрийн storage эх үүсвэрүүдийг нэг интерфэйс дор нэгтгэх боломж олгодог. Гэсэн хэдий ч ийм өргөн хүрээтэй боломжууд нь системийг суулгах, тохируулах, удирдах процессыг төвөгтэй болгож, энгийн хэрэглэгчдэд хүндрэл учруулдаг.

\paragraph{}
Нөгөө талаас Syncthing нь peer-to-peer зарчимд суурилсан файл синхрончлолын систем бөгөөд төв сервер шаардалгүйгээр төхөөрөмжүүдийг хооронд нь шууд холбодог. Syncthing нь Local Discovery, Global Discovery, Relay Protocol зэрэг механизмуудыг \cite{syncthingspecifications} ашиглан сүлжээний янз бүрийн орчинд төхөөрөмжүүдийг илрүүлэх, холбох боломжийг олгодог. Гэвч Syncthing нь олон төхөөрөмж дээр нэг фолдерийг хуулбарлан хадгалах (replication) зарчимд суурилсан тул уламжлалт cloud storage-ийн өгөгдөл нэг газарт төвлөрсөн байдлаар хадгалагдана гэсэн ойлголттой нийцдэггүй. 
\paragraph{}
Мөн хэрэглэгчийн хүлээлтийг тодорхойлогч consumer cloud storage системүүд болох Apple iCloud болон Google Drive нь автомат backup, background upload, минимал тохиргоо зэрэг UX-ийн өндөр стандартыг тогтоосон. Эдгээр системүүд нь хэрэглээний хувьд энгийн боловч өгөгдөл нь гуравдагч талын төв серверт хадгалагддаг.

\subsection{Peer-to-Peer өгөгдөл синхрончлолын системүүд}

Peer-to-peer (P2P) зарчимд суурилсан өгөгдөл синхрончлолын системүүд:
\begin{enumerate}
    \item Syncthing (opensource)
    \item Resilio Sync
\end{enumerate}

Эдгээр системүүд нь төхөөрөмж бүрийг давтагдашгүй таних тэмдэг (device identity)-ээр тодорхойлж, төв сервер шаардахгүйгээр хооронд нь шууд холбох боломжийг олгодог. Төхөөрөмжүүдийг илрүүлэхдээ дотоод сүлжээнд LAN discovery механизмыг ашиглах бөгөөд шаардлагатай тохиолдолд relay fallback \cite{syncthingspecifications} шийдлийг ашиглан холболтыг үргэлжлүүлэх боломжтой. Мөн өгөгдөл дамжуулалт нь end-to-end encryption (E2EE)-д суурилсан zero-trust загвараар хамгаалагддаг нь аюулгүй байдлын чухал давуу тал болдог.

Гэсэн хэдий ч эдгээр системүүд нь төхөөрөмжүүдийн хооронд файлыг харилцан хуулбарлан хадгалах (replication) зарчимд суурилдаг тул өгөгдөл нэг төв байршилд хадгалагдах уламжлалт хувийн cloud storage загвараас ялгаатай. Иймээс тэдгээрийн хэрэглээний зорилго нь бүрэн нийцэхгүй боловч дотооддоо ашиглаж буй холболт, илрүүлэлт болон аюулгүй дамжуулалтын технологиуд нь судлан хэрэгжүүлэхэд чухал ач холбогдолтой юм.

\subsection{Харьцуулсан шинжилгээ}



\section{Ашиглах технологиуд}

\paragraph{}
Сервер талын програм хангамжийг Go програмчлалын хэл дээр хэрэгжүүлнэ. Go нь Google компанийн хөгжүүлсэн компиляцлагддаг програмчлалын хэл бөгөөд C болон C++ хэлүүдийн нэгэн адил машин код руу шууд хөрвүүлэгддэг. Иймээс interpreted хэлнүүд болох Python, JavaScript зэрэгтэй харьцуулахад бага тооцооллын нөөц (CPU, RAM) ашиглан өндөр гүйцэтгэлтэй ажиллах боломжийг олгодог.

Go хэл нь сервер талын програм хангамж хөгжүүлэхэд онцгойлон зориулагдсан орчин үеийн хэл бөгөөд энгийн синтакс, хөнгөн runtime орчин, болон нэг бинар файл хэлбэрээр програмыг хөрвүүлэх боломж зэрэг давуу талуудтай. Энэ нь серверийг суулгах болон байршуулж ажиллуулах үйл явцыг хялбар болгодог.

Мөн Go хэл нь зэрэгцээ програмчлал (concurrency)-ийг goroutine болон channel механизмын тусламжтайгаар дэмждэг. Goroutine нь маш хөнгөн жинтэй гүйцэтгэлийн нэгж бөгөөд сервер олон хэрэглэгчийн хүсэлтийг зэрэг боловсруулах боломжийг олгодог. Үүний үр дүнд систем нь олон хэрэглэгч нэгэн зэрэг хандах үед ч тогтвортой, үр ашигтай ажиллах боломжтой.

\paragraph{}
Хэрэглэгчийн талын аппликейшнийг хоёр төрлөөр хөгжүүлнэ. Үүнд iOS үйлдлийн системд зориулсан native аппликейшн болон веб аппликейшн орно.

iOS аппликейшнийг Swift програмчлалын хэл болон SwiftUI framework ашиглан хөгжүүлнэ. Swift нь Apple компанийн хөгжүүлсэн орчин үеийн програмчлалын хэл бөгөөд iOS платформд зориулсан өндөр гүйцэтгэлтэй, найдвартай програм хөгжүүлэхэд өргөн хэрэглэгддэг.

Веб аппликейшнийг Angular framework ашиглан хөгжүүлнэ. Angular нь TypeScript дээр суурилсан орчин үеийн веб framework бөгөөд Single Page Application (SPA) архитектурыг хэрэгжүүлэх боломжийг олгодог. Энэ нь хэрэглэгч веб хөтөч ашиглан сервер дээр хадгалагдсан файлуудыг харах, удирдах боломжийг бүрдүүлнэ.

\section{Бүлгийн дүгнэлт}
\paragraph{}
