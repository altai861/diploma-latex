    \section*{Удиртгал}
        Өнөөгийн дижитал эрин үед үүлэн хадгалалт (Cloud Storage) нь хэрэглэгчдийн өдөр тутмын амьдралын салшгүй хэсэг болжээ. Хэрэглэгчид зураг, бичлэг, ажлын болон хувийн чухал файлуудаа интернетийн тусламжтайгаар хаанаас ч хандаж ашиглах шаардлагатай байдаг. Энэхүү хэрэгцээг хангахын тулд Google Drive, iCloud, OneDrive зэрэг төвлөрсөн үүлэн хадгалалтын үйлчилгээ өргөн хэрэглэгддэг болсон.

        Гэвч эдгээр үйлчилгээний хувьд хэрэглэгчийн өгөгдөл тухайн үйлчилгээ үзүүлэгч компанийн сервер дээр хадгалагддаг бөгөөд хэрэглэгч өөрийн мэдээллийн бүрэн эзэмшил, хяналтыг алдах эрсдэлтэй байдаг. Мөн хадгалах багтаамж нэмэгдэх тусам тогтмол төлбөр төлөх шаардлага гардаг нь урт хугацаанд хэрэглэгчдэд эдийн засгийн дарамт учруулж болзошгүй.

        Иймд хэрэглэгч өөрийн эзэмшлийн техник хангамжийг ашиглан өгөгдлөө хадгалах боломжтой, өөрөөр хэлбэл өөрийн сервер дээр суурилсан хувийн үүлэн хадгалалтын системийг хөгжүүлэх нь чухал асуудал болж байна. Энэхүү ажлын хүрээнд хэрэглэгчийн өгөгдлийг өөрийн сервер дээр хадгалах, веб болон мобайл клиентээр дамжуулан удирдах боломжтой, хөнгөн бүтэцтэй (lightweight) хувийн үүлэн хадгалалтын системийг хөгжүүлэхийг зорьж байна.
    \subsection*{Зорилго}
    Энэхүү төслийн зорилго нь хэрэглэгч өөрийн эзэмшлийн техник хангамжид суурилсан, өөрөө байршуулж (self-hosted) ажиллуулах боломжтой хувийн үүлэн хадгалалтын системийг зохион бүтээж хэрэгжүүлэхэд оршино. Тус систем нь хэрэглэгчдэд веб болон мобайл клиент ашиглан өөрийн сервер дээр хадгалагдсан файлуудыг удирдах, хандах боломжийг олгоно.

    \subsection*{Төслийн зорилтууд}
        Дэвшүүлсэн зорилгод хүрэхийн тулд дараах зорилтуудыг хэрэгжүүлнэ. Үүнд:

        \begin{enumerate}[nosep]
            \item Хувийн үүлэн хадгалалтын системүүдийн архитектур болон технологийн харьцуулсан судалгаа хийх;
            \item Нөөц багатай төхөөрөмж (Raspberry Pi зэрэг) дээр ажиллах боломжтой хөнгөн серверийн программ хангамжийг зохион бүтээх;
            \item Файл хадгалах, татах, зохион байгуулах үндсэн үйлдлүүдийг дэмжих серверийн REST API хөгжүүлэх;
            \item Хэрэглэгч веб хөтөч ашиглан файлуудаа удирдах боломжтой веб клиент аппликейшн хөгжүүлэх;
            \item iOS үйлдлийн системд зориулсан хэрэглэгчийн мобайл аппликейшн боловсруулах;
            \item Хэрэглэгч өөрийн серверийг интернэт орчинд байршуулан алсаас хандах боломжийг бүрдүүлэх архитектурын шийдлийг боловсруулах.
        \end{enumerate}

    \subsection*{Судлах үндэслэл}
        Сүүлийн жилүүдэд үүлэн хадгалалтын хэрэглээ огцом өсөж, 2025 оны байдлаар дэлхий даяар үүлэн хадгалалтын хэрэглэгчдийн тоо 2.3 тэрбумыг \cite{cloudstorageusage} давсан байна. Гэвч төвлөрсөн үүлэн үйлчилгээний хэрэглээ өсөхийн хэрээр дараах асуудлууд тулгарч байна.

        \begin{enumerate}
            \item \textbf{Өгөгдлийн эзэмшил ба нууцлал}: Хэрэглэгчийн мэдээлэл гуравдагч этгээдийн сервер дээр хадгалагддаг тул өгөгдлийн бүрэн хяналт хэрэглэгчид байхгүй.
            \item \textbf{Тогтмол үйлчилгээний зардал}: Үүлэн хадгалалтын үйлчилгээний багтаамж нэмэгдэх тусам хэрэглэгч тогтмол төлбөр төлөх шаардлагатай болдог.
        \end{enumerate}

        Иймд хэрэглэгч өөрийн техник хангамжийг ашиглан өгөгдлөө хадгалах боломжтой, өөрөөр хэлбэл self-hosted хувийн үүлэн системийг хөгжүүлэх нь технологийн болон практик ач холбогдолтой юм.

    \subsection*{Судлагдсан байдал}
        Хувийн үүлэн хадгалалтын системийн чиглэлээр Nextcloud, Seafile, Syncthing зэрэг нээлттэй эхийн төслүүд өргөн хэрэглэгддэг. Эдгээр системүүд нь хэрэглэгчийн өгөгдлийг өөрийн сервер дээр хадгалах боломжийг олгодог боловч ихэвчлэн том хэмжээний системийн архитектуртай, олон төрлийн нэмэлт функцүүдийг агуулсан байдаг.

        Тухайлбал, Nextcloud нь хамтын ажиллагаа, календарь, мэйл зэрэг олон төрлийн үйлчилгээг нэг дор багтаасан цогц платформ бол Seafile нь файл синхрончлолын өндөр гүйцэтгэлд төвлөрсөн систем юм. Гэвч эдгээр системүүдийг бага нөөцтэй төхөөрөмж дээр ажиллуулахад тохиргоо хийхэд төвөгтэй, системийн шаардлага өндөр байх зэрэг сул талууд ажиглагддаг.

        Иймд энэхүү төслөөр илүү хөнгөн бүтэцтэй, хувийн хэрэглээнд зориулсан үүлэн хадгалалтын системийг зохион бүтээхийг зорьсон.
        
   \subsection*{Шинэлэг тал}
    \textbf{Хувийн үүлэн хадгалалтын системийн шинэлэг тал:} 
        \begin{enumerate}[itemsep=0mm]
            \item \textbf{Хөнгөн архитектур}: Нөөц багатай төхөөрөмжүүд дээр ажиллах боломжтой энгийн бөгөөд үр ашигтай серверийн архитектур ашиглана.
            \item \textbf{Self-hosted шийдэл}: Хэрэглэгч өөрийн сервер дээр системийг байршуулж өгөгдлөө бүрэн хянах боломжтой.
            \item \textbf{Олон төрлийн клиент}: Веб болон мобайл клиент ашиглан файлуудад хандах боломжтой.
        \end{enumerate}

    \subsection*{Технологийн ач холбогдол}
         Энэхүү систем нь технологийн хувьд дараах ач холбогдолтой:
         \begin{enumerate}[itemsep=0mm]
            \item \textbf{Өгөгдлийн бүрэн эзэмшил}: Хэрэглэгч өөрийн сервер дээр өгөгдлөө хадгалснаар мэдээллийн бүрэн хяналтыг хадгална.
            \item \textbf{Хөнгөн серверийн шийдэл}: Орчин үеийн асинхрон серверийн архитектур ашиглан өндөр гүйцэтгэлтэй систем бий болгоно.
            \item \textbf{Олон платформын клиент}: Веб болон мобайл клиентүүдээр дамжуулан хэрэглэгчийн хүртээмжийг нэмэгдүүлнэ.
        \end{enumerate}

    \subsection*{Хамрах хүрээ}
    Тус төсөл нь дараах хүрээг хамарна:
    \begin{itemize}[nosep]
        \item Өөрийн өгөгдлийн нууцлалыг эрхэмлэдэг хувь хүн болон гэр бүлийн хэрэглэгчид;
        \item Өндөр өртөг бүхий үүлэн үйлчилгээнээс татгалзаж, өөрийн нөөцийг ашиглах хүсэлтэй технологи сонирхогчид;
    \end{itemize}
