\section{Хэрэгжүүлсэн системийн ерөнхий бүтэц}
\paragraph{}
Гуравдугаар бүлэгт тодорхойлсон зохиомжид тулгуурлан хувийн үүлэн хадгалалтын системийг үндсэн сервер, веб клиент, iOS клиент болон алсын холболтод зориулсан дамжуулагч сервер гэсэн бүрэлдэхүүнтэйгээр хэрэгжүүлэв. Үндсэн сервер нь хэрэглэгчийн өөрийн төхөөрөмж дээр ажиллаж, хэрэглэгчийн бүртгэл, нэвтрэлт, файл болон хавтас удирдах, хуваалцах эрхийн логик, хадгалалтын мета өгөгдөл болон бодит файлын зохион байгуулалтыг хариуцна. Веб болон iOS клиентүүд нь хэрэглэгчийн өдөр тутмын файл удирдах урсгалыг нэг ижил үйлчилгээний API-р дамжуулан ашиглана.

\paragraph{}
Дотоод сүлжээнд байгаа үед клиентүүд үндсэн сервертэй шууд холбогдох байдлаар ажиллана. Харин хэрэглэгч өөр сүлжээнээс хандах шаардлагатай үед үндсэн сервер болон дамжуулагч серверийн хооронд үүсгэсэн холболтоор дамжуулан хүсэлтүүдийг хэрэглэгчийн төхөөрөмж рүү хүргэх зохион байгуулалт ашигласан. Ингэснээр хэрэглэгчийн файлууд гадаад сервер дээр хадгалагдахгүй, үндсэн өгөгдөл өөрийн төхөөрөмж дээр үлдэх зарчим хадгалагдана.

\section{Нэвтрүүлэлтийн орчин}
\paragraph{}
Үндсэн серверийн аппликейшнийг Raspberry Pi төхөөрөмж дээр байршуулж ажиллуулах байдлаар туршсан. Энэ орчин нь системийн зорилго болох бага нөөцтэй, хэрэглэгчийн өөрийн эзэмшлийн төхөөрөмж дээр ажиллах боломжийг шалгах үндсэн суурь болсон. Дамжуулагч серверийг public VPS орчинд байршуулснаар дотоод сүлжээнээс гадна байгаа хэрэглэгч өөрийн төхөөрөмж дээрх серверт хандах урсгалыг турших боломж бүрдсэн.

\paragraph{}
Суурилуулалтын үед серверийн тохиргоо, өгөгдлийн сангийн холболт, хадгалалтын үндсэн зам, төхөөрөмжийн таних нэр болон алсын холболтын тохиргоог тус тус шалгасан. Мөн серверийг systemd service байдлаар ажиллуулснаар төхөөрөмж дахин ассан үед үйлчилгээ автоматаар эхлэх боломжийг баталгаажуулсан.

\section{Системийн туршилт}
\paragraph{}
Системийн баталгаажуулалтыг үндсэн хэрэглээний урсгалууд дээр төвлөрүүлэн хийсэн. Үүнд анхны тохиргоо хийх, хэрэглэгч нэвтрэх, файл болон хавтас жагсаах, файл байршуулах, татах, нэр өөрчлөх, устгах, сэргээх, мөн хэрэглэгч хооронд нөөц хуваалцах үйлдлүүдийг шалгасан. Эдгээр туршилт нь хоёрдугаар бүлэгт тодорхойлсон функциональ шаардлагууд бодит систем дээр хэрэгжиж буй эсэхийг баталгаажуулах зорилготой.

\begin{enumerate}
    \item \textbf{Функциональ туршилт:} Нэвтрэх, файл байршуулах, татах, хавтас үүсгэх, нэр өөрчлөх, устгах, сэргээх зэрэг өдөр тутмын хэрэглээний урсгалуудыг шалгасан.
    \item \textbf{Эрхийн туршилт:} Админ болон энгийн хэрэглэгчийн эрхийн ялгаа, эзэмшигч болон засварлагч эрхийн хязгаарлалт, зөвшөөрөлгүй хандалтыг хориглох логикийг шалгасан.
    \item \textbf{Интерфейсийн туршилт:} Веб болон мобайл клиент дээр үндсэн дэлгэцүүдийн харагдах байдал, хэрэглээний дараалал, алдааны мэдэгдэл ойлгомжтой эсэхийг шалгасан.
    \item \textbf{Холболтын туршилт:} Дотоод сүлжээнд шууд холбогдох болон гадаад сүлжээнээс алсаар хандах үндсэн урсгалуудыг шалгасан.
\end{enumerate}

\paragraph{}
Туршилтын үр дүнд системийн үндсэн файл удирдлагын урсгалууд ажиллах боломжтой нь баталгаажсан. Мөн бага нөөцтэй төхөөрөмж дээр байршуулж ажиллуулах үед серверийн үндсэн үйлдлүүд хэвийн ажиллаж, веб болон мобайл клиентүүд нэг ижил backend үйлчилгээтэй холбогдон ажиллах боломжтой байв.

\chapterconclusiontitle{Бүлгийн дүгнэлт}
\paragraph{}
Энэ бүлэгт гуравдугаар бүлэгт тодорхойлсон зохиомжийг бодит систем болгон хэрэгжүүлсэн байдал, хэрэглэгчийн интерфейсийн бодит хувилбарууд, нэвтрүүлэлтийн орчин болон туршилтын үр дүнг тайлбарлалаа. Үндсэн серверийг хэрэглэгчийн өөрийн төхөөрөмж дээр ажиллуулж, веб болон мобайл клиентээр дамжуулан файл удирдах үндсэн урсгалуудыг шалгаснаар системийн зорилго болох self-hosted хувийн үүлэн хадгалалтын үндсэн боломжууд хэрэгжих боломжтойг харууллаа.
