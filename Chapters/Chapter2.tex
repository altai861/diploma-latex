\section{Системийн үйл ажиллагааны тухай дэлгэрэнгүй}

\paragraph{}
Хэрэглэгч өөрийн хувийн компьютерт хувийн үүлэн хадгалалтын серверийн программыг суулгаж ажиллуулна. Мөн урьдчилан тохируулсан бэлэн төхөөрөмжийг ашиглах боломжтой. Анхны тохиргооны үе шатанд систем нь тухайн төхөөрөмжид хамаарах давтагдашгүй таних мэдээллийг автоматаар үүсгэнэ. Үүний дараа хэрэглэгч үүлэн хадгалалтад ашиглах үндсэн фолдерыг сонгон тохируулна. Ингэснээр сервер талын бэлтгэл хангагдана. Дараагийн алхамд хэрэглэгч гар утсандаа мобайл клиент аппликейшнийг татан суулгаж, анхны нэвтрэлтийг хийнэ.

\paragraph{}
Сервер болон мобайл төхөөрөмжийн хооронд анхны холболтыг үүсгэхийн тулд хоёр төхөөрөмж нэг дотоод сүлжээнд холбогдсон байх шаардлагатай. Энэ үед судалгааны хэсгийн \ref{sec:lan-detection}-т тайлбарласан дотоод сүлжээн дэх төхөөрөмж илрүүлэх механизмыг ашиглан сервер автоматаар илрүүлэгдэнэ. Мобайл аппликейшний хэрэглэгчийн интерфэйс дээр боломжит серверүүдийн жагсаалт харагдах бөгөөд хэрэглэгч өөрийн серверийг сонгон холболтыг эхлүүлнэ.

Холболтын хүсэлт илгээгдсэний дараа сервер талд тухайн мобайл төхөөрөмжийн мэдээлэл харагдаж, хэрэглэгч зөвшөөрөх эсвэл татгалзах шийдвэр гаргана. Хэрэв зөвшөөрөл олгогдвол холболт амжилттай үүсэж, тухайн мобайл төхөөрөмж сервертэй бүртгэгдэнэ. Ийнхүү бүртгэгдсэн төхөөрөмж нь хэрэглэгч гараар устгах хүртэл сервертэй дахин холбогдох боломжтой байна.

\paragraph{}
Холболт тогтсоны дараа систем нь төхөөрөмжүүд ижил дотоод сүлжээнд байгаа эсэхийг автоматаар тодорхойлно. Хэрэв хоёр төхөөрөмж нэг Wi-Fi сүлжээнд байрлаж байвал шууд локал (LAN) холболт үүсгэж, өгөгдөл дамжуулалтыг дотоод сүлжээгээр гүйцэтгэнэ. Харин өөр өөр сүлжээнд байрласан тохиолдолд интернетээр дамжуулан алсын холболтыг автоматаар хэрэгжүүлнэ.

Амжилттай холболт үүссэний дараа хэрэглэгч гар утаснаасаа сервер дээр байрлах үүлэн хадгалалтын фолдерын бүтэц болон файлуудыг харах, шаардлагатай файлыг татаж авах, шинэ файл байршуулах, файл болон хавтсыг засварлах, устгах, зохион байгуулах зэрэг үйлдлийг гүйцэтгэх боломжтой болно. Ийнхүү хэрэглэгч өөрийн өгөгдлийг хувийн компьютерт төвлөрүүлэн хадгалж, хандалт болон удирдлагыг бүрэн хяналтандаа байлгах нөхцөл бүрдэнэ.

\section{Функциональ шаардлага}

\begin{enumerate}
    
    \item Үүлэн хадгалалтад ашиглах үндсэн фолдерыг сонгон тохируулах боломжтой байх.
    
    \item Хэрэглэгч гар утсан дээрээ дотоод сүлжээнд байгаа серверүүдийн жагсаалт харах боломжтой байна.
    
    \item Гар утаснаас холбогдох хүсэлтийг илгээх боломжтой байх.

    \item Сервер дээр холбогдох хүсэлтийг зөвшөөрөх болон татгалзах боломжтой байх.
    
    \item Үүлэн хадгалалтын сервертэй холбогдсон гар утсыг серверээс устгах боломжтой байх.
    
    \item Гар утаснаас серверийн холболтоо устгах боломжтой байх.
    
    \item Төхөөрөмжүүдийн сүлжээний байршлаас хамааран локал эсвэл алсын холболтыг автоматаар сонгодог байх.
    
    \item Сервер дээр хадгалагдсан файл, хавтасын бүтцийг харах боломжтой байх.
    
    \item Файл байршуулах, татаж авах, устгах болон зохион байгуулах боломжтой байх.
\end{enumerate}

\section{Функциональ бус шаардлага}

\begin{enumerate}
    \item Серверийн программ нь Linux, Windows, MacOS үйлдлийн системүүд дээр амжилттай сууж ажиллах боломжтой байх.

    \item Мобайл аппликейшн нь iOS үйлдлийн систем дээр ажиллах боломжтой байх.

    \item Гар утас болон сервер хоёрын хооронд бүх өгөгдөл E2EE ашиглан шифрлэгдэж дамжуулдаг байх.
\end{enumerate}

\section{Ажлын хүрээ}

\subsection{Scope (Хамрах хүрээ)}

Энэхүү төслийн хүрээнд дараах програм хангамжийн бүрэлдэхүүнүүдийг хөгжүүлнэ:

\begin{enumerate}
    \item \textbf{Хувийн үүлэн хадгалалтын үндсэн серверийн программ (Main Server Application): } Үндсэн серверийн программ нь хэрэглэгчийн компьютерт суурилж, өгөгдлийг хадгалах, төхөөрөмж бүртгэх, холболтыг удирдах, файл дамжуулалтыг гүйцэтгэх үүрэгтэй байна.
    \item \textbf{Алсын холболтыг дэмжих relay серверийн программ (Relay Server Application): }Relay сервер нь төхөөрөмжүүд өөр сүлжээнд байрласан тохиолдолд дамжуулалтын зуучлагч байдлаар ажиллах бөгөөд өгөгдлийг хадгалахгүй, зөвхөн шифрлэгдсэн мэдээллийг дамжуулах үүрэгтэй байна.
    \item \textbf{iOS үйлдлийн системд зориулсан мобайл клиент аппликейшн: } iOS мобайл аппликейшн нь сервертэй холбогдох, файл удирдах, өгөгдөл татах болон байршуулах боломжийг хангана.
\end{enumerate}

\subsection{Out of Scope (Хийхгүй зүйлс)}

\begin{enumerate}

    \item \textbf{Веб аппликейшн: }
    Энэхүү төслийн ажлын хүрээнд веб аппликейшн хөгжүүлэхгүй болно. 
    Учир нь веб орчинд дотоод сүлжээнд сервер автоматаар илрүүлэх 
    механизмыг хэрэгжүүлэхэд хязгаарлалттай бөгөөд төслийн үндсэн 
    шаардлагад багтах end-to-end шифрлэлтийг (E2EE) найдвартай 
    хэрэгжүүлэхэд нэмэлт төвөгшил үүсэхээр байна. 
    Иймд веб хувилбарыг цаашдын хөгжүүлэлтийн боломжит чиглэл 
    хэмээн үзэж, энэхүү ажлын хүрээнд оруулахгүй болно.

    \item \textbf{Android мобайл аппликейшн: }

    Мөн Android үйлдлийн системд зориулсан мобайл аппликейшн 
    энэхүү төслийн хүрээнд хөгжүүлэхгүй. Төслийн хүрээнд 
    нэг платформ дээр төвлөрч, native хөгжүүлэлтийг гүнзгий түвшинд 
    хийх зорилгоор iOS платформыг сонгосон болно.

    \item \textbf{Олон хэрэглэгчийн эрхийн менежмент: }

    Энэхүү төслийн хүрээнд олон хэрэглэгчийн эрхийн менежментийн
    (Role-based access control) систем хөгжүүлэхгүй. 
    Системийн баталгаажуулалт нь хэрэглэгчийн бүртгэлд бус,
    төхөөрөмжийн түвшний таних болон зөвшөөрлийн механизмд
    суурилсан байна.

\end{enumerate}

\section{Хөгжүүлэлтийн төлөвлөгөө}

\paragraph{}
Төслийн ажлыг долоо хоногт 16 цаг зарцуулна гэж тооцоолон,
урьдчилсан хамгаалалт хүртэлх 12 долоо хоногийн хугацаанд
ажлуудыг үе шаттайгаар төлөвлөсөн. Ажлын задаргаа (WBS)-д
тодорхойлсон багц ажлуудыг долоо хоног бүрт хуваарилан,
хамаарал бүхий ажлуудын дарааллыг харгалзан гүйцэтгэх
хуваарь боловсруулав. Энэхүү төлөвлөлтийн дагуу нийт
192 цагийн ажлыг 12 долоо хоногт багтаан хэрэгжүүлэхээр
тооцсон болно.

\subsection{Долоо хоног бүрийн зорилтууд}

\begin{enumerate}
    \item \textbf{1-р долоо хоног (03-04 - 03-11): Суурь бүтцийг байгуулах.} \\
    Үндсэн серверийн болон iOS төслийн анхны суурь бүтэц (2.1, 3.1)-ийг байгуулж ажиллуулах, мөн системийн архитектурын баримтжуулалтыг (1.1) эхлүүлэх.

    \item \textbf{2-р долоо хоног (03-11 - 03-18): Аюулгүй байдлын суурь ба pairing-ийн эхлэл.} \\
    Үндсэн серверийн түлхүүрийн удирдлага, шифрлэлтийн логикийг (2.3) хэрэгжүүлэх, төхөөрөмж холбох (pairing) ба баталгаажуулалтын механизмыг (2.2) эхлүүлэх, архитектурын баримт бичгийг (1.1) дуусгах.

    \item \textbf{3-р долоо хоног (03-18 - 03-25): LAN дээр холболтыг бүрэн ажиллуулах.} \\
    Pairing/auth механизмыг (2.2) дуусгах, iOS талын LAN илрүүлэлт ба сервертэй холболтын урсгалыг (3.2) хэрэгжүүлэх, iOS талын түлхүүр/шифрлэлтийн логикийг (3.3) хэрэгжүүлэх.

    \item \textbf{4-р долоо хоног (03-25 - 04-01): Дотоод сүлжээний демо бэлтгэх.} \\
    Файл дамжуулалтын үндсэн боломжуудыг хангах demo түвшний File API (2.4) болон iOS File UI (3.4)-г хэрэгжүүлж, дотоод сүлжээнд ажиллах демог бэлэн болгох.

    \item \textbf{5-р долоо хоног (04-01 - 04-08): Файл ажиллагааг өргөтгөж гүйцээх.} \\
    File API (2.4)-г бүрэн гүйцээж, iOS File UI (3.4)-г сайжруулах, мөн туршилт ба баталгаажуулалтын хэсгийн бичвэрийг (1.2) эхлүүлэх.

    \item \textbf{6-р долоо хоног (04-08 - 04-15): Дамжуулагч серверийн суурь ба сервер--relay холболтын эхлэл.} \\
    Дамжуулагч серверийн суурь бүтцийг (4.1) байгуулах, үндсэн серверээс relay руу холбогдох логикийг (2.5) хэрэгжүүлж эхлүүлэх.

    \item \textbf{7-р долоо хоног (04-15 - 04-22): Дамжуулагч серверийн routing ба session management.} \\
    Relay дээр серверүүдийн хоорондын routing логикийг (4.2) хэрэгжүүлэх, гар утаснаас ирэх холболтуудын session боловсруулалтыг (4.3) хэрэгжүүлж эхлүүлэх.

    \item \textbf{8-р долоо хоног (04-22 - 04-29): Дамжуулагч серверийн bridge ба remote урсгал ажиллуулах.} \\
    Relay bridge логикийг (4.4) хэрэгжүүлэх, iOS талын relay ашиглан холбогдох логикийг (3.5) хэрэгжүүлж, remote холболтын үндсэн урсгалыг ажиллуулах.

    \item \textbf{9-р долоо хоног (04-29 - 05-06): Дамжуулагч серверийг нэгтгэж, remote холболтыг тогтвортой болгох.} \\
    Relay талын интеграцчилал, туршилт, алдаа засварыг (4.5) хийх, байршуулалтын ажлыг (4.6) гүйцэтгэх, үндсэн серверийн интеграцчилал/алдаа засварыг (2.6) эхлүүлэх.

    \item \textbf{10-р долоо хоног (05-06 - 05-13): Системийн интеграцчилал ба тогтворжилт.} \\
    Үндсэн серверийн интеграцчилал, тест, алдаа засварыг (2.6) үргэлжлүүлэх, iOS талын интеграцчилал ба bug fixing (3.6)-г гүйцэтгэх, мөн тайлангийн туршилт/баталгаажуулалтын хэсгийг (1.2) үргэлжлүүлэх.

    \item \textbf{11-р долоо хоног (05-13 - 05-20): Тайлангийн үндсэн хэсгүүдийг дуусгах.} \\
    Удиртгал (1.5), дүгнэлт (1.4), refactoring-той холбоотой бичвэрүүд (1.3)-ийг гүйцээж, тайлангийн үндсэн агуулгыг нэгтгэн дуусгах.

    \item \textbf{12-р долоо хоног (05-20 - 05-27): Эцсийн шалгалт ба хамгаалалтын бэлтгэл.} \\
    Эцсийн засвар, форматлал, зураг/хүснэгтийн шалгалт, үлдсэн алдаа засварыг хийх, мөн бүтэн демог (12-р долоо хоногийн төгсгөл) тогтвортой байдлаар бэлэн болгох.
\end{enumerate}

\section{Бүлгийн дүгнэлт}

Энэхүү бүлэгт системийн хэрэглэгчийн дэлгэрэнгүй урсгалыг тодорхойлж, үүн дээр үндэслэн системийн функциональ шаардлагын жагсаалтыг боловсруулав. Мөн системийн чанарын үзүүлэлтүүдийг тодорхойлох функциональ бус шаардлагуудыг тодорхойлсон болно. Үүнээс гадна системийн use case диаграммыг боловсруулж, төслийн ажлын хамрах болон хамрахгүй хүрээг тодорхойлон, 12 долоо хоногийн ажлын төлөвлөгөөг боловсруулан танилцууллаа.
