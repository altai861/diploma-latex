\section{Системийн үйл ажиллагааны дэлгэрэнгүй}

\paragraph{}
Энэ бүлэгт тодорхойлох шаардлагуудыг зөвхөн серверийн програм хангамжийн түвшинд бус, хэрэглэгчийн өөрийн эзэмшлийн төхөөрөмж дээр ажиллах хувийн үүлэн хадгалалтын үйлчилгээний хүрээнд авч үзнэ. Өөрөөр хэлбэл систем нь Raspberry Pi зэрэг бага нөөцтэй төхөөрөмж дээр ажиллах үндсэн сервер, түүнд хандах веб болон iOS клиент, дотоод сүлжээнд хялбар илрүүлэх механизм, мөн шаардлагатай үед гадаад сүлжээнээс холбогдох боломжуудыг хамарна.

\paragraph{}
Хэрэглэгч системээ гэрийн болон байгууллагын дотоод сүлжээнд ашиглах үед серверийг автоматаар илрүүлж шууд холбогдох боломжтой байна. Харин хэрэглэгч өөр сүлжээнд, жишээлбэл мобайл дата эсвэл гадны Wi-Fi ашиглаж байгаа үед өөрийн хувийн үүлэн сервертээ алсаас хандах боломж шаардлагатай. Энэ тохиолдолд систем нь хэрэглэгчээс нарийн сүлжээний тохиргоо шаардахгүйгээр төхөөрөмжийн таних мэдээлэлд тулгуурлан холбогдох хувилбарыг дэмжинэ. Гэхдээ алсын холболт ашигласан үед ч хэрэглэгчийн файл болон үндсэн өгөгдөл нь өөрийн төхөөрөмж дээр хадгалагдах зарчим хэвээр байна.

\paragraph{}
Энэхүү системийг хоёр тусдаа ажиллах хэсэгтэй байхаар төлөвлөнө. Нэг нь хэрэглэгчийн өдөр тутмын файл удирдах үйлдлүүдийг хариуцах үндсэн хэсэг байх бөгөөд нөгөө нь зөвхөн анхны тохиргоонд зориулагдсан тусгай хэсэг байна. Систем анхны тохиргоо хийгдээгүй үед зөвхөн анхны тохиргооны хэсэг ажиллах бөгөөд энэ зохион байгуулалт нь эхний тохиргооны эмзэг үйлдлүүдийг ердийн хэрэглээний урсгалаас салгана.

\paragraph{}
Анхны тохиргооны үеийн шаардлагад систем анхны тохиргоо хийгдсэн эсэхийг шалгах боломж орно. Үүний дараа эхний админ хэрэглэгчийн мэдээллийг оруулах бөгөөд үүнд хэрэглэгчийн нэр, и-мэйл, бүтэн нэр, нууц үг, нууц үгийн баталгаажуулалт, мөн өгөгдөл хадгалах үндсэн зам багтана. Нэмэлтээр системийн нийт хадгалалтын дээд хэмжээ, төхөөрөмжийн нэр, дотоод сүлжээнд зарлагдах хаяг зэрэг төхөөрөмжид хамаарах үндсэн тохиргоог тодорхойлох боломжтой байна. Тохиргоо амжилттай дууссаны дараа шаардлагатай хадгалалтын бүтэц үүсэж, анхны тохиргооны хэсэг цаашид идэвхгүй болохоор зохион байгуулна.

\paragraph{}
Анхны тохиргооны дараа хэрэглэгч веб аппликейшн эсвэл iOS аппликейшнээр дамжин нэвтрэх нэр, нууц үг ашиглан системд нэвтрэх боломжтой байна. Дотоод сүлжээнд байгаа үед клиент нь серверийг автоматаар илрүүлэх эсвэл зарлагдсан локал хаягаар холбогдох боломжтой байна. Харин хэрэглэгч гадаад сүлжээнд байгаа үед өөрийн төхөөрөмжийг сонгож, хувийн үүлэн сервертээ алсаас хандах боломжтой байхаар төлөвлөнө. Амжилттай нэвтэрсэн тохиолдолд систем хэрэглэгчийг баталгаажуулж, цаашдын үйлдлүүдийг гүйцэтгэх эрх олгоно.

\paragraph{}
Админ хэрэглэгч нь системийн удирдлагын хэсгээр дамжин хэрэглэгчдийн бүртгэлийг удирдах шаардлагатай. Тэрээр бүх хэрэглэгчийг жагсааж харах, шинэ энгийн хэрэглэгч үүсгэх, хэрэглэгчийн үндсэн мэдээлэл болон хадгалалтын хязгаарыг шинэчлэх, мөн админ биш хэрэглэгчийг устгах боломжтой байна. Хэрэглэгч устах үед тухайн хэрэглэгчийн нэвтрэлтийн төлөв, профайлын зураг, хуваалцсан эрхүүд болон хувийн хадгалалтын бүтэц хамтад нь цэвэрлэгдэхээр шаардлага тодорхойлно.

\paragraph{}
Файлын сангийн логикийг ``өөрийн эзэмшлийн орчин'' болон ``өөртэй нь хуваалцсан орчин'' гэсэн хоёр үндсэн хэсэгтэй байна. Хэрэглэгч өөрийн хавтас, файлуудыг шаталсан бүтэцээр харах, дэлгэрэнгүй мэдээлэл авах, хандах эрхтэй өгөгдлүүд дотроос хайлт хийх, мөн устгасан файлууд, онцолсон файлууд, хуваалцсан өгөгдөл гэсэн тусгай харагдацуудаар ажиллах боломжтой байх ёстой. Ингэснээр өдөр тутмын хэрэглээний үндсэн урсгалууд нэгтгэсэн зохион байгуулалттай байна.

\paragraph{}
Үйлдлийн эрхийн хувьд эзэмшигч болон засварлагч эрхийг ялгаж үзэхээр төлөвлөнө. Засварлагч эрхтэй хэрэглэгч хуваалцсан хавтас дотор хавтас үүсгэх, файл байршуулах, зарим нэр өөрчлөх үйлдэл хийх боломжтой байх боловч эрхийн удирдлага, шилжүүлэх, устгах, сэргээх, онцлох зэрэг эзэмшигч төвтэй үйлдлүүдийг хийхгүй. Харин эзэмшигч хэрэглэгч өөрийн файл, хавтсан дээр бусдад харах эсвэл засварлах эрх олгох, устгасан өгөгдлөө сэргээх, бүр мөсөн устгах, мөн олон файл, хавтсыг нэг дор шахсан файл хэлбэрээр татах зэрэг өргөн удирдлагын боломжтой байна.

\section{Функциональ шаардлага}

\begin{enumerate}

\item Эхний суурилуулалтын үе шатанд админ хэрэглэгчийн мэдээлэл, системийн ашиглах багтаамжийн зам, хэмжээ, төхөөрөмжийн нэр болон дотоод сүлжээнд зарлагдах үндсэн тохиргоог хийнэ.

\item Хэрэглэгч нэвтрэх нэр болон нууц үгээ ашиглан системд нэвтэрнэ.

\item Клиент аппликейшн нь дотоод сүлжээнд байгаа хувийн үүлэн серверийг автоматаар илрүүлэх эсвэл зарлагдсан локал хаягаар холбогдох боломжтой байна.

\item Хэрэглэгч гадаад сүлжээнээс хандах үед өөрийн хувийн серверийг сонгож, түүнтэй алсаас холбогдох боломжтой байна.

\item Хэрэглэгч өөрийн төхөөрөмжийг системд танигдах нэр эсвэл таних кодоор ялган харах боломжтой байна.

\item Хэрэв хэрэглэгчийн хувийн сервер тухайн үед холбогдох боломжгүй байвал клиент аппликейшн ойлгомжтой алдааны мэдээлэл харуулах боломжтой байна.

\item Админ бүх хэрэглэгчийн жагсаалтыг харах боломжтой байна.

\item Админ шинэ энгийн хэрэглэгч үүсгэхдээ нэвтрэх нэр, и-мэйл, бүтэн нэр, нууц үг, хадгалалтын хязгаар зэрэг мэдээллийг оруулна.

\item Админ хүссэн хэрэглэгчийнхээ мэдээллийг шинэчлэх боломжтой байна.

\item Админ энгийн хэрэглэгчийг устгах боломжтой байна.

\item Хэрэглэгч өөрийн болон өөртэй нь хуваалцсан файл, хавтаснуудыг харах боломжтой байна. Харахдаа төрөл, нэр, засварлагдсан он сар өдрөөр ангилах боломжтой байна.

\item Хэрэглэгч устгасан файлууд, онцолсон файлууд, хуваалцсан нөөц гэсэн тусгай харагдацуудаар өгөгдлөө харах боломжтой байна.

\item Хэрэглэгч өөрийн хандах эрхтэй файл, хавтаснуудаас нэрээр хайлт хийх боломжтой.

\item Хэрэглэгч файл болон хавтсын дэлгэрэнгүй мэдээллийг харах боломжтой байна.

\item Эзэмшигч эсвэл засварлагч эрхтэй хэрэглэгч зөвшөөрөгдсөн хавтас дотор шинэ хавтас үүсгэх боломжтой байна.

\item Эзэмшигч эсвэл засварлагч эрхтэй хэрэглэгч зөвшөөрөгдсөн хавтас дотор файл байршуулах боломжтой байна.

\item Эзэмшигч эсвэл засварлагч эрхтэй хэрэглэгч зөвшөөрөгдсөн файл, хавтсын нэрийг өөрчлөх боломжтой байна.

\item Эзэмшигч хэрэглэгч өөрийн файл, хавтсуудыг өөрийн эзэмшлийн өөр хавтас руу шилжүүлэх боломжтой байна.

\item Эзэмшигч хэрэглэгч өөрийн файл, хавтсыг устгасан файлын хэсэг рүү зөөх, тэндээс сэргээх, мөн бүр мөсөн устгах боломжтой байна.

\item Хэрэглэгч өөрийн хандах эрхтэй файлыг татаж авах, урьдчилан харах, мөн хэд хэдэн файл, хавтсыг нэг дор шахсан файл хэлбэрээр татах боломжтой байна.

\item Эзэмшигч хэрэглэгч өөрийн файл, хавтас дээр бусад хэрэглэгчдэд харах эсвэл засварлах эрх олгох, эрхийн жагсаалтыг харах, мөн тухайн эрхийг цуцлах боломжтой байна.

\item Хэрэглэгч өөрийн эзэмшлийн файл, хавтсыг онцолсон төлөвт оруулах болон тухайн төлөвөөс гаргах боломжтой байна.

\end{enumerate}


\section{Функциональ бус шаардлага}

\begin{enumerate}

\item Серверийн программ нь Raspberry Pi, жижиг Linux сервер, эсвэл хэрэглэгчийн өөрийн техник хангамж дээр байршуулан ажиллуулах боломжтой, бага нөөцийн хэрэглээтэй байх.

\item Систем нь LAN-first зарчимтай байх бөгөөд клиент болон сервер нэг дотоод сүлжээнд байгаа үед өгөгдлийг гадаад сүлжээгээр тойруулалгүй шууд холбогдох боломжтой байх.

\item Хэрэглэгчийн client аппликейшн нь web болон iOS платформ дээр ажиллах боломжтой байх.

\item Web аппликейшн нь Chrome, Firefox, Safari болон Edge браузеруудын сүүлийн хоёр хувилбар дээр зөв ажиллах боломжтой байх.

\item Хэрэглэгч болон серверийн хооронд дамжих бүх өгөгдөл аюулгүй сувгаар шифрлэгдсэн байх.

\item Алсын холболт ашигласан үед ч хэрэглэгчийн өгөгдлийн эзэмшил өөрт нь хэвээр байх бөгөөд файл хадгалалтын үндсэн орчин нь хэрэглэгчийн өөрийн төхөөрөмж байх.

\item Алсын холболтын боломж нь хэрэглэгчийн хувьд ойлгомжтой, нэмэлт нарийн сүлжээний тохиргоо аль болох бага шаардах байдлаар төлөвлөгдөх.

\item Дотоод болон алсын холболтын аль ч үед файл байршуулах, татах үндсэн урсгал хэрэглэгчийн хувьд ижил мэдрэмжтэй байх.

\item Системийг цааш өргөтгөхдөө илүү олон хэрэглэгч, олон төхөөрөмж, олон зэрэг холболтыг дэмжих боломжийг харгалзан төлөвлөх.

\item Хэрэглэгчийн нууц үгийг сервер дээр plaintext хэлбэрээр хадгалахгүй бөгөөд аюулгүй hash алгоритм ашиглан хадгалдаг байх.

\item Систем нь бага нөөцтэй төхөөрөмж дээр хэд хэдэн хэрэглэгч зэрэг хандах үед үндсэн үйлдлүүдийн зөв ажиллагааг алдагдуулахгүй байх.

\item Файл байршуулах болон татах үйлдлийн явцыг хэрэглэгчийн интерфэйс дээр харуулах боломжтой байх.

\item Систем нь хэрэглэгч зөвхөн өөрийн файл болон өөрт олгогдсон эрхтэй өгөгдөлд хандах боломжийг хангах байх.

\item Файл дамжуулах болон хадгалах үед өгөгдлийн бүрэн бүтэн байдлыг шалгах боломжтой байх.

\item Системийн суурилуулалт, анхны тохиргоо, хадгалалтын зам сонгох үйл явц нь энгийн хэрэглэгчид аль болох ойлгомжтой байх.

\end{enumerate}

\section{Класс диаграм}

\paragraph{}
Usecase-үүдийг дараагийн шатанд дарааллын диаграммаар илэрхийлэх боломжтой байлгахын тулд энэ классын диаграммыг \textit{boundary-control-entity} бүтэцтэйгээр боловсруулсан. Зураг \ref{fig:analysis-class-diagram}-т системийн гаднах оролт хүлээн авах boundary классууд, үндсэн логикийг зохицуулах control классууд, мөн домэйн өгөгдлийг илэрхийлэх entity классуудын хамаарлыг харуулав.

\begin{figure}[H]
    \centering
    \includegraphics[width=\textwidth]{Figures/chapter2/design-class-diagram.jpg}
    \caption{Хувийн үүлэн хадгалалтын системийн класс диаграм}
    \label{fig:analysis-class-diagram}
\end{figure}

\paragraph{}
Диаграммын дээд хэсэгт  \textit{SetupBoundary}, \textit{AuthBoundary}, \textit{StorageBoundary}, \textit{ShareBoundary}, \textit{AdminBoundary} классуудыг байрлуулсан. Эдгээрийг системтэй харилцах үндсэн оролтын цэгүүд гэж үзсэн бөгөөд анхны тохиргоо, нэвтрэлт, хадгалалтын үйлдэл, хуваалцах урсгал, админы үйлдэл зэрэг usecase-үүдийн эхлэл хэсгийг төлөөлөхөөр сонгосон.

\paragraph{}
Boundary классуудын доор  \textit{SetupController}, \textit{AuthenticationController}, \textit{StorageController}, \textit{SharingController}, \textit{UserManagementController} гэсэн control классуудыг байрлуулсан. Эдгээр нь usecase-ийн гол логикийг хариуцаж, boundary болон entity давхаргын хооронд зуучлах үүрэгтэй байхаар загварчилсан. Жишээлбэл \textit{StorageController}-т файл, хавтастай холбоотой үндсэн урсгалуудыг, \textit{SharingController}-т хуваалцах эрхийн логикийг, \textit{AuthenticationController}-т нэвтрэлт ба session удирдлагыг төвлөрүүлэн оруулсан.

\paragraph{}
\textit{Entity} давхаргад  \textit{UserEntity}, \textit{StorageEntity}, \textit{FolderEntity}, \textit{FileEntity}, \textit{SessionEntity}, \textit{SystemConfigEntity}, \textit{SharePermissionEntity} гэсэн үндсэн классуудыг оруулсан. Энд \textit{StorageEntity}-г хадгалалтын нийтлэг шинжийг нэгтгэсэн суурь класс байдлаар тодорхойлж, түүнээс \textit{FolderEntity} болон \textit{FileEntity} өвлөгдөхөөр зохиосон. Харин \textit{SharePermissionEntity} болон \textit{AccessLevel} enum-аар нөөц хуваалцах эрхийн түвшинг, \textit{UserRole} enum-аар хэрэглэгчийн үүргийг илэрхийлсэн. Иймээс энэхүү диаграммыг  системийн бүтцийг ерөнхий түвшинд харуулахын зэрэгцээ дарааллын диаграмм байгуулахад оролцогч классуудыг тодорхойлох суурь болгон ашигласан.

\section{Ажлын хүрээ}

Энэхүү төслийн хүрээнд хэрэглэгчийн өөрийн төхөөрөмж дээр ажиллах хувийн үүлэн хадгалалтын үйлчилгээний дараах бүрэлдэхүүнүүдийг хөгжүүлнэ:

\begin{enumerate}
    \item \textbf{Хувийн үүлэн хадгалалтын үндсэн серверийн программ: } Үндсэн сервер нь Raspberry Pi зэрэг хэрэглэгчийн өөрийн төхөөрөмж дээр байршиж ажиллах бөгөөд системийн анхны тохиргоо, нэвтрэлт, хэрэглэгчийн удирдлага, файл болон хавтас харах, байршуулах, татах, үүсгэх, устгах, шилжүүлэх, мөн хуваалцах эрхийн удирдлага зэрэг үндсэн боломжуудыг хангана.
    \item \textbf{Дамжуулагч серверийн программ: } Хэрэглэгчийг интернетээр дамжин хувийн системдээ хандах боломжоор хангана. 
    \item \textbf{iOS үйлдлийн системд зориулсан мобайл клиент аппликейшн: } iOS мобайл аппликейшн нь өдөр тутмын хэрэглэгчийн үндсэн урсгалуудыг дэмжинэ. Үүнд системд нэвтрэх, өөрийн файл болон хавтсыг харах, файл байршуулах, татах, хавтас үүсгэх, өгөгдөл устгах, шилжүүлэх, өөрт хуваалцсан өгөгдөлтэй ажиллах зэрэг боломжууд орно.
    \item \textbf{Веб аппликейшн: } Веб аппликейшн нь энгийн хэрэглэгчийн үндсэн урсгалуудыг дэмжихийн зэрэгцээ анхны тохиргоо хийх, хэрэглэгчийн жагсаалт харах, хэрэглэгч үүсгэх, мэдээлэл шинэчлэх, хэрэглэгч устгах зэрэг админы урсгалуудыг мөн дэмжинэ.
\end{enumerate}

\chapterconclusiontitle{Бүлгийн дүгнэлт}
\paragraph{}
Энэ бүлэгт хувийн үүлэн хадгалалтын системийн үндсэн шаардлагуудыг usecase, функциональ шаардлага, функциональ бус шаардлага, мөн шинжилгээний класс диаграммын түвшинд тодорхойллоо. Үүний үр дүнд хөгжүүлэх систем нь анхны тохиргоо, нэвтрэлт, хэрэглэгчийн удирдлага, файл болон хавтастай ажиллах үндсэн үйлдлүүд, хуваалцах эрхийн удирдлагаас гадна бага нөөцтэй төхөөрөмж дээр ажиллах, дотоод сүлжээнд хялбар илрэх, шаардлагатай үед алсаас хандах боломжтой байхаар төлөвлөгдөж байна. Мөн эзэмшигч, засварлагч, админ хэрэглэгчийн ялгаа, өөрийн төхөөрөмж дээр ажиллах орчны хязгаарлалт, хөнгөн жинтэй байх шаардлага нь системийн зохиомжид шууд нөлөө үзүүлнэ.

\paragraph{}
Өөрөөр хэлбэл нэгдүгээр бүлэгт хийсэн судалгааны үр дүнг энэ бүлэгт системийн шаардлага болон өгөгдлийн бүтцийн түвшинд буулгасан гэж үзэж болно. Дараагийн бүлэгт эдгээр шаардлагад тулгуурласан системийн архитектур, өгөгдлийн сангийн зохиомж, интерфэйсийн шийдлүүдийг дэлгэрэнгүй тайлбарлана.
